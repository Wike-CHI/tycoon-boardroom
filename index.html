The Process

讨论怎么进行

三轮推进：先定性，再拆解，最后收敛成你能立刻执行的动作。

1

问题定性

由最敏锐的成员复述你的核心问题，用他自己的语言重新描述。通常一开口就让你意识到："对，这才是我的真问题。"

2

多角度剖析

五位大佬从各自视角分析，每人引用自己的失败经历。允许反驳、插话、沉默。至少出现一次明显的意见分歧。

3

收敛方案

由务实派总结 1-3 条明天就能执行的具体行动建议。每条是确定的动作，不是方向性描述。

Role Chemistry

角色化学反应

不是各说各话。他们之间有自然的联盟和冲突，讨论会真实地碰撞。

🤝 马化腾 + 雷军

务实落地派 — 都重视效率和用户。一个从产品角度拆解，一个用数据和定价落地。

🤝 马云 + 马斯克

颠覆派 — 都敢于打破常规。一个从组织使命出发，一个从第一性原理出发。

⚖️ 黄仁勋

架构裁判 — 经常让其他人的争论显得"格局小了"。把短期问题拉到10年维度重新评估。

⚡ 愿景 vs 渐进

马云的"All in" vs 马化腾的"小步快跑" — 要不要赌上一切？

⚡ 原创 vs 学习

马斯克的"回到原理" vs 雷军的"学习最佳实践" — 自己造还是抄最好的？

Live Preview

讨论长什么样

一段真实的讨论片段，感受五位大佬的化学反应。

  tycoon-boardroom output

会议室。窗外是深圳湾的夜景。五个人围坐在一张长桌旁。

👤 马化腾 （推了推眼镜，低头想了两秒）

我觉得先把问题说清楚。你现在做的不是"选什么技术栈"的问题——你是在选你要成为什么样的人。21岁，最大的资本是时间，但你把它花在同时做五件事上。

👤 马云 （身体前倾，手指点向空气）

我告诉你，你问错问题了！你不是在问"做什么"，你在问"怎么不痛"。我当年被30家风投拒绝的时候，也没有备选方案。但你得先想清楚——你到底信不信你自己？

👤 雷军 （拍了下桌子）

说真的，我不同意Jack说的"不要备选方案"。小米2015年销量暴跌36%，我当时就是想得太大了。创始人必须下沉到一线——你先选一件事，做到极致。

👤 马斯克

Stop。你们都在讨论"做什么"。Wrong question。

（沉默了五秒，盯着远处）

他的问题不是选择太多，是每一件事都做得不够深入。你需要的不是砍掉选项，是找到那个一旦做成其他都不重要的事。

👤 黄仁勋 （微笑摇头）

你们想得太小了。他21岁就在做AI——这不是一个职业选择问题，这是一个架构问题。你们2006年要是问我要不要做CUDA，所有人都说我疯了。但你把底层架构搭对了，应用层自然会来。

Get Started

安装和使用

这是一个 Claude Code Skill，安装只需要一个命令。

  macOS / Linux 
curl -fsSL https://raw.githubusercontent.com/Wike-CHI/tycoon-boardroom/main/install.sh | bash
  Windows PowerShell 
irm https://raw.githubusercontent.com/Wike-CHI/tycoon-boardroom/main/install.ps1 | iex
1

运行上面的命令，一键安装全部技能

自动下载 tycoon-boardroom 及其依赖的马化腾、雷军、比尔·盖茨档案到 ~/.claude/skills/

2

在 Claude Code 中触发

输入 /tycoon-boardroom 然后描述你的问题，或在对话中说"让大佬们帮我看看"

3

拿到方案，明天就做

讨论结束时会给你 1-3 条具体的行动建议，附带优先级。
